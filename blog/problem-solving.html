Problem Solving

When I was blocked on a simple problem

In the JavaScript-Kata challenge I spent so much time on Manipulating objects specifically the second point

// 2. Give the adam object a name key that has the value "Adam".

adam = { Name: “Adam” }

Which wasn't correct. And it took a lot of research on doing dot notation for objects. So I found out using

adam.name = "Adam";

That creates a new property in the Adam object. Which passed the test!

When I elegantly solved a problem

Doing the javascript cafe last sprint. Doing the stretch exercises.

I found a way that changes the text to red if it's out of stock. I struggled at first but after doing some research I found you can change The inner html text colour by using style.color = “Colour you want here”

I was able to use a condition that checked if the current stock was in stock And if it wasn't then change the text to red. Otherwise change it back to black.

//if its out of stock change text to red if(!products.whiteCoffee.stock){ document.getElementById("whiteCoffee").style.color = '#d00'

//if its already red change it back to black text because we now have it } else if (products.whiteCoffee.stock){ document.getElementById("whiteCoffee").style.color = '#000000' }

Pseudocode:

I found Pseudocode to be very helpful because you can write out roughly how you want to solve a problem. Your plan can change but the core of the pseudocode is there. Its like freestyling a recipe. You have a rough guideline but you can add and change things on the fly.

Trying something
